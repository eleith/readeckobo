# readeckobo

`readeckobo` is a Go application that acts as a bridge between Kobo e-readers and a [Readeck](https://readeck.com) instance. It emulates the Instapaper API to allow Kobo devices to sync articles from Readeck.

This project is a Go port of the original Python-based [kobeck](https://github.com/Lukas0907/kobeck).

## Features

*   Fetches and lists un-archived articles from Readeck for Kobo devices.
*   Downloads article content in a Kobo-compatible format.
*   Handles actions like archiving, re-adding, favoriting, and deleting articles by updating Readeck.
*   Converts images to JPEG format on the fly for better e-reader compatibility.
*   Supports multiple Kobo devices with different tokens.

## Configuration

The application is configured using a `config.yaml` file. An example is provided in `config.yaml.example`:

```yaml
server:
  port: 8080
log_level: info
readeck:
  host: "https://your-readeck-instance.com"
users:
  - token: "a-very-secret-token-for-your-kobo"
    readeck_access_token: "your-plain-text-readeck-access-token"
  - token: "another-secret-token-for-another-device"
    readeck_access_token: "your-other-readeck-access-token"
```

-   `server.port`: The port on which the `readeckobo` server will run.
-   `log_level`: The logging level. `info` is recommended for normal use. `debug` is useful for troubleshooting.
-   `readeck.host`: The URL of your Readeck instance.
-   `users`: A list of users or devices.
    -   `token`: A secret token that you will configure on your Kobo device. This acts as a password.
    -   `readeck_access_token`: Your plain-text Readeck access token.

### Generating a Device Token

To generate a new random token for your device, you can use the `generate-token` script located in the `bin` directory:

```sh
./bin/generate-token
```

On your Kobo device, you will use this generated token as the password for the Instapaper account.

## Security Considerations

The authentication model of this application is designed for simplicity and assumes you are running it in a trusted environment. Here are some important security considerations:

*   **HTTPS is strongly recommended.** Always run `readeckobo` behind a reverse proxy that provides HTTPS. This will protect your device tokens and Readeck access tokens from being transmitted in plain text over the network.
*   **Network Exposure:** For the best security, do not expose the `readeckobo` server to the public internet. Keep it on your local network, accessible only to your Kobo device.
*   **Device Security:** Anyone with physical access to your Kobo device can potentially access your Readeck account. It is recommended to set a password or PIN on your Kobo device to prevent unauthorized access.
*   **Token Revocation:** If you lose a device or a token is compromised, you should immediately remove the corresponding user entry from your `config.yaml` and restart the `readeckobo` server.

## Usage

### Building and Running Locally

1.  **Install dependencies:**
    ```sh
    go mod tidy
    ```

2.  **Build the application:**
    ```sh
    make build
    ```

3.  **Run the application:**
    ```sh
    ./readeckobo
    ```

### Testing with E2E Scripts

The `scripts/e2e-tests/` directory contains simple shell scripts to test each API endpoint. These are useful for verifying that your `readeckobo` setup and connection to Readeck are working correctly.

Each script requires parameters, such as your device token. Run a script without arguments to see its usage instructions.

Example:
```sh
# Make sure the scripts are executable
chmod +x scripts/e2e-tests/*.sh

# Run the 'get' test
./scripts/e2e-tests/01-test-get.sh <YOUR_DEVICE_TOKEN>
```

### Using the Makefile

The `Makefile` provides several useful targets:

-   `make build`: Build the application binary.
-   `make test`: Run all unit tests.
-   `make lint`: Run the linter.
-   `make vendor`: Vendor all dependencies.
-   `make ci`: Run all CI checks (linting and testing).

### Running with Docker

To run the application using Docker:

1.  **Build the Docker image:**
    ```sh
    docker-compose build
    ```

2.  **Run the container:**
    ```sh
    docker-compose up
    ```

The server will be available at `http://localhost:8080`.
